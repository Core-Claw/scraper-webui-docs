Worker tasks allow you to create and save **multiple preset configurations** for the same Worker, making it easy to reuse and run them on demand.

For example, you can configure a **Product Scraping** task to extract product information, reviews, and other data from Amazon. Once the target URLs and parameters are set, the same configuration can be reused to scrape different products.

You can manage and run these task configurations directly in the **CoreClaw Console**, as well as view their execution history, enabling efficient task management and reuse.
