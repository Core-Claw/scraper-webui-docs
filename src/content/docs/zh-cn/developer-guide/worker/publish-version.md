## 什么是发布版本？

在 CoreClaw 平台中，每个脚本（Script）在更新或修改后都可以生成一个新的发布版本（Version）。发布版本用于：

- 记录脚本更新历史
- 保证线上执行的稳定性
- 便于用户选择指定版本运行

每次正式发布都会生成一个独立的版本编号。

## 如何发布新版本？

1. 登录控制台
2. 进入脚本管理页面
3. 编辑脚本内容或配置
4. 点击「发布」按钮
5. 系统生成新的版本编号

发布完成后，新任务默认使用最新版本（如未指定版本）。

### 版本管理说明

平台通常支持以下操作：

- 查看历史版本
- 选择指定版本运行任务
- 回滚到历史稳定版本
- 删除草稿版本（如适用）

:::tip[最佳实践]
建议在重大修改前保留稳定版本，以便快速恢复。
:::

### 注意事项

- 修改脚本内容不会自动影响已运行任务
- 仅发布审核后的版本可用于正式执行
- 不同版本可能对应不同的计费或资源消耗情况
