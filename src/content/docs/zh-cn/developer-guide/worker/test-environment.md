### 什么是测试环境？

测试环境是一个独立运行环境，用于：

- 验证抓取参数
- 调试数据字段
- 检查反爬情况
- 优化抓取逻辑
- 预览数据结构

该环境**不影响生产任务数据**，也不覆盖正式运行记录。

### 测试环境与正式环境的区别

| 项目 | 测试环境 | 正式环境 |
|------|---------|---------|
| 数据规模 | 小规模样本 | 全量数据 |
| 运行目的 | 调试与验证 | 正式采集 |
| 执行频率 | 临时运行 | 可定时运行 |
| 数据保存 | 短期保存 | 按平台规则保存 |
| 费用消耗 | 通常低消耗或限制消耗 | 按实际资源消耗计费 |

:::note[建议]
在正式部署前，务必先在测试环境中验证运行结果。
:::

### 如何使用测试环境？

#### 步骤 1：进入 Script 页面

1. 登录个人账户
2. 进入控制台
3. 点击"我的Script"
4. 选择需要测试的 Script

#### 步骤 2：配置测试参数

- 在输入参数区域，建议使用默认参数。

#### 步骤 3：查看测试结果

测试完成后可查看：

- 数据预览
- 字段结构
- 执行日志（Logs）
- 错误信息

确认数据结构正确后，再进行正式运行。

### 注意事项

- 测试环境仅用于调试，不建议用于正式数据交付
- 测试数据可能仅保留有限时间
- 测试运行同样可能消耗少量资源
- 若频繁失败，请查看日志信息或联系技术支持
