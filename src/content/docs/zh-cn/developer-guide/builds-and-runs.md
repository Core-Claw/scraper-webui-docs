了解如何在 CoreClaw 上管理 Worker 构建和运行。

## Worker 构建

构建是将 Worker 代码和依赖打包为平台可运行单元的过程。

### 构建流程

1. **代码上传** - Worker 代码以 ZIP 压缩包形式上传
2. **依赖安装** - 平台安装所需的包
3. **环境准备** - 准备脚本运行环境
4. **就绪状态** - Worker 准备运行

### 构建状态

| 状态 | 说明 |
| ---- | ---- |
| **PENDING** | 构建排队中 |
| **BUILDING** | 构建进行中 |
| **SUCCESS** | 构建成功完成 |
| **FAILED** | 构建遇到错误 |

### 管理构建

- 在 Worker 详情页**查看所有构建**
- **检查构建日志**进行调试
- **重新构建**以重试失败的构建

## Worker 运行

运行是已构建 Worker 的单次执行。每次运行在轻量级的进程隔离脚本运行环境中执行。

### 运行生命周期

```
READY → RUNNING → SUCCEEDED/FAILED/ABORTING
```

### 运行状态

| 状态 | 说明 |
| ---- | ---- |
| **READY** | 运行排队等待中 |
| **RUNNING** | 运行正在执行 |
| **SUCCEEDED** | 运行成功完成 |
| **FAILED** | 运行遇到错误 |
| **ABORTING** | 运行正在停止 |

### 运行管理

- **监控** - 实时查看运行进度
- **中止** - 停止正在运行的 Worker
- **重新运行** - 使用相同参数执行
- **查看日志** - 调试执行问题
- **下载结果** - 获取输出数据

## 运行历史

CoreClaw 会保留您所有的 Worker 运行记录。每次运行都会保存：

| 字段 | 说明 |
| ---- | ---- |
| **运行 ID** | 唯一标识符 |
| **状态** | 当前状态 |
| **时长** | 执行时间 |
| **输入** | 使用的参数 |
| **输出** | 结果（如果成功） |
| **日志** | 执行日志 |

## 最佳实践

### 构建

- 保持依赖最小化
- 锁定依赖版本
- 上传前在本地测试

### 运行

- 从小规模测试运行开始
- 监控日志发现问题
- 使用任务进行重复运行
