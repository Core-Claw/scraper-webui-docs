每个 CoreClaw 套餐都允许**同时**执行的 Worker 运行数量存在上限。这样可以在不同用户之间公平分配执行资源，并保证平台稳定。该上限针对**同时活跃**的运行数量，而不是你总共可以运行多少次。

## 各套餐的并发运行数

| 套餐 | 最大并发运行数 |
| ---- | ------------: |
| Free | 2 |
| Starter | 5 |
| Pro | 10 |
| Business | 50 |

## 哪些运行会占用额度

只有当前正在执行的运行才会占用并发额度。已经结束的运行——无论成功、失败还是被取消——都不占用。

| 运行状态 | 是否占用额度 |
| -------- | ------------ |
| `running` | 是 |
| `succeeded` | 否 |
| `failed` | 否 |
| `aborting` / 已取消 | 否 |

也就是说：

```
已占用并发数 = 当前处于 running 状态的运行数量
```

## 达到上限时

当你已经达到套餐上限却仍要启动新运行时，平台不会创建该运行，并会提示类似信息：

> **暂时无法启动运行**
> 同时执行的运行数量已达到套餐上限，请等待正在运行的任务完成后再试。

因该原因被拒绝时：

- 不创建运行（不生成 Run ID）
- 不扣除余额
- 不消耗运行次数

请等待某个正在运行的任务结束后再重新启动。

## 释放额度

当运行离开 `running` 状态时（即达到 `succeeded`、`failed`，或被你取消后），会立即释放其并发额度。取消运行后，一旦运行离开 `running`，其额度会立即释放。

## 变更套餐

- **升级** —— 更高的上限立即生效，新运行按新上限执行。
- **降级** —— 正在执行的运行不受影响、继续运行，只有新运行按更低的上限执行。
