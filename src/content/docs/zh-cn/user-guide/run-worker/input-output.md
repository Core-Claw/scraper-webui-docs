了解如何配置 Worker 输入并理解输出数据结构。

## 输入参数

每个 Worker 接受特定的输入参数来控制其行为。这些参数在 Worker 的**输入配置**中定义。

### 常见输入类型

| 类型 | 说明 | 示例 |
| ---- | ---- | ---- |
| **string** | 文本输入 | URL、搜索关键词 |
| **number** | 数字输入 | 页数限制、超时 |
| **boolean** | 真/假开关 | 是否包含图片 |
| **array** | 值列表 | 多个 URL |
| **object** | 嵌套配置 | 代理设置 |

### 配置输入

运行 Worker 时，您会看到基于其输入配置的表单：

1. **必填字段**必须在运行前填写
2. **可选字段**有默认值
3. **帮助文本**解释每个参数的用途

## 输出数据

Worker 输出是 JSON 格式的结构化数据。结构取决于 Worker 的用途。

### 输出结构

```json
{
  "data": [
    {
      "title": "产品名称",
      "price": 99.99,
      "url": "https://example.com/product"
    }
  ],
  "metadata": {
    "total": 100,
    "scrapedAt": "2024-01-15T10:30:00Z"
  }
}
```

### 导出格式

您可以以以下格式导出输出数据：

| 格式 | 适用场景 |
| ---- | -------- |
| **JSON** | 开发者、API 集成 |
| **CSV** | 电子表格、数据分析 |

## 数据大小限制

- 大数据集会分页
- 使用 API 流式传输大结果
