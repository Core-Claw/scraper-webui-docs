了解 Worker 如何在 CoreClaw 平台上运行。

## 无需构建——轻量脚本模式

CoreClaw 的运行流程是：

```
用户代码 → 脚本运行环境 → 远程指纹浏览器池(CDP) → 目标网站
```

CoreClaw 采用的是**轻量级进程隔离**，非容器隔离：

- **运行环境是平台预置的**：Python/Node 运行时、基础依赖已经安装在共享环境中。
- **浏览器是远程托管的**：不需要在本地构建浏览器环境，而是通过 CDP 连接远程指纹浏览器池。
- **依赖安装是自动的**：平台根据 `requirements.txt` / `package.json` 自动安装，不需要用户手动构建镜像。

所以用户只需要：**上传脚本 / 商店找到想要使用的脚本 → 点击运行**，平台自动处理依赖安装和运行。

**简单来说**：CoreClaw 没有构建步骤，是因为它把运行环境和浏览器都做了**平台级托管**，用户不需要关心环境构建——这是它的优势：**上传即运行**。

## 运行生命周期

```
READY → RUNNING → SUCCEEDED / FAILED / ABORTING
```

| 状态 | 说明 |
| ---- | ---- |
| **READY** | 运行排队等待中 |
| **RUNNING** | 脚本正在执行 |
| **SUCCEEDED** | 运行成功完成 |
| **FAILED** | 运行遇到错误 |
| **ABORTING** | 运行正在被停止 |

## 运行管理

- **监控** - 实时查看运行进度
- **中止** - 停止正在运行的 Worker
- **重新运行** - 使用相同参数再次执行
- **查看日志** - 调试执行问题

## 运行历史

CoreClaw 会保留您所有的 Worker 运行记录。每次运行都会保存：

- 唯一运行 ID
- 时间戳
- 时长
- 状态
- 输入参数
- 输出数据
- 日志

您可以在控制台处查看运行记录。
