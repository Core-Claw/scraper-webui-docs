了解 Worker 如何在 CoreClaw 平台上构建和执行。

## Worker 构建

Worker 运行前，必须在平台上完成构建。构建过程将 Worker 代码和依赖打包为可运行单元。

### 构建流程

1. **代码上传** - Worker 代码以 ZIP 压缩包形式上传
2. **依赖安装** - 平台安装所需的包
3. **环境准备** - 准备脚本运行环境
4. **就绪状态** - Worker 准备运行

### 构建状态

| 状态 | 说明 |
| ---- | ---- |
| **PENDING** | 构建排队中 |
| **BUILDING** | 构建进行中 |
| **SUCCESS** | 构建成功完成 |
| **FAILED** | 构建遇到错误 |

### 构建日志

查看构建日志以调试构建失败：

- 依赖错误
- 语法错误
- 配置问题

## Worker 运行

运行是已构建 Worker 的单次执行。每次运行在轻量级的进程隔离脚本运行环境中执行。

### 运行生命周期

```
READY → RUNNING → SUCCEEDED/FAILED/ABORTING
```

### 运行管理

- **监控** - 实时查看运行进度
- **中止** - 停止正在运行的 Worker
- **重新运行** - 使用相同参数执行
- **查看日志** - 调试执行问题

## 运行历史

CoreClaw 会保留您所有的 Worker 运行记录。每次运行都会保存：

- 唯一运行 ID
- 时间戳
- 时长
- 状态
- 输入参数
- 输出数据
- 日志
