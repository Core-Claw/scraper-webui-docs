了解如何诊断和修复常见 Worker 失败问题。

## 理解失败状态

Worker 运行可能因各种原因失败。运行状态会显示 **FAILED** 及错误信息。

## 常见失败原因

### 1. 输入验证错误

**症状：**
- 运行立即失败
- 错误信息提到"无效输入"

**解决方案：**
- 检查必填字段是否填写
- 验证 URL 格式
- 确保数值在范围内

### 2. 目标网站问题

**症状：**
- 运行开始但在执行过程中失败
- 超时错误

**解决方案：**
- 验证网站是否可访问
- 检查网站是否已更改
- 尝试启用 Web Unlocker

### 3. 反爬检测

**症状：**
- 空结果
- 验证码错误
- IP 被封消息

**解决方案：**
- 启用 Web Unlocker
- 使用代理轮换
- 降低请求频率

### 4. 资源限制

**症状：**
- 内存不足错误
- 超时错误

**解决方案：**
- 减少范围（更少页面）
- 拆分为多次运行
- 联系支持升级资源

### 5. 账户问题

**症状：**
- 余额不足
- 权限被拒绝

**解决方案：**
- 充值账户余额
- 检查订阅状态
- 验证 API 密钥权限

## 调试步骤

### 步骤 1：检查运行日志

进入运行详情页面并查看日志：

```
[ERROR] 连接超时 30 秒
[ERROR] 无法解析来自 https://example.com 的响应
```

### 步骤 2：验证输入参数

仔细检查您的输入配置：

- URL 正确且可访问
- 必填字段已填写
- 值在预期范围内

### 步骤 3：使用最小输入测试

使用最简单的配置运行：

- 使用单个 URL 而非多个
- 最少页面
- 默认设置

### 步骤 4：联系支持

如果问题持续：

- 📧 邮箱：support@coreclaw.com
- 包含：运行 ID、错误信息、输入参数

## 预防技巧

| 问题 | 预防措施 |
| ---- | -------- |
| 输入错误 | 运行前验证 |
| 超时 | 使用适当的超时设置 |
| 反爬 | 启用 Web Unlocker |
| 频率限制 | 在请求之间添加延迟 |
