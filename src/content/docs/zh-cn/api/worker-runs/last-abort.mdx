## 什么时候使用这个接口

用于中止仍可中止的运行。调用前应确认目标运行。


## 请求示例

```bash
curl -X POST "https://openapi.coreclaw.com/api/v2/worker-runs/last/abort" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## 响应示例

```json
{
  "code": 0,
  "message": "success",
  "request_id": "req-123"
}
```

## 注意事项

- API v2 同时支持 Bearer token、旧版 `api-key` 请求头和 query token；新集成建议优先使用 Bearer token。
- 这是账户级最近运行接口，不需要 `workerId` 路径参数；它会作用于当前账户范围内最近一次运行。
- `last` 表示当前筛选范围内最近一次运行；如需确定目标，请使用具体 `runId` 接口。
- 运行状态、轮询、失败诊断和取消处理请参阅[运行生命周期与状态](/zh-cn/api/run-lifecycle/)。
- 取消请求成功只表示服务端已接受取消操作；随后应针对同一具体 `runId` 读取详情。不要等待或自行构造未在契约中定义的 `aborted` 状态。
