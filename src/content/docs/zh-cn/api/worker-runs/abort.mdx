**方法:** `POST`

**端点:** `/api/v2/worker-runs/{runId}/abort`

**认证:** 支持 `Authorization: Bearer <YOUR_API_KEY>`、`api-key: <YOUR_API_KEY>` 和 `?token=<YOUR_API_KEY>`。推荐优先使用 Bearer token。

## 什么时候使用这个接口

用于中止仍可中止的运行。调用前应确认目标运行。

## 标识符说明

- `runId` 是运行记录 slug。启动或重跑后的 `data.run_slug` 就是后续接口使用的 `runId`。

## 路径参数

| 参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| `runId` | 是 | `string` | 运行记录 slug，即启动或重跑响应中的 `data.run_slug`。 |

## 请求示例

```bash
curl -X POST "https://openapi.coreclaw.com/api/v2/worker-runs/YOUR_RUN_ID/abort" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## 响应示例

```json
{
  "code": 0,
  "message": "success",
  "request_id": "req-123"
}
```

## 注意事项

- API v2 同时支持 Bearer token、旧版 `api-key` 请求头和 query token；新集成建议优先使用 Bearer token。
- 运行状态、轮询、失败诊断和取消处理请参阅[运行生命周期与状态](/zh-cn/api/run-lifecycle/)。
- 取消请求成功只表示服务端已接受取消操作；随后应针对同一具体 `runId` 读取详情。不要等待或自行构造未在契约中定义的 `aborted` 状态。

## HTTP 响应

| HTTP 状态 | 应用代码 | 含义 |
| --- | --- | --- |
| `200` | `0` | 请求成功。 |
| `400` | `11000` | 请求参数不合法。 |
| `401` | `12001` | 认证缺失或无效。 |
| `404` | `11004` | 目标资源不存在。 |
| `422` | `11000` | 请求语义或字段校验未通过。 |
| `429` | `13000` | 请求过于频繁。 |
| `500` | `10000` | 服务端内部错误。 |
