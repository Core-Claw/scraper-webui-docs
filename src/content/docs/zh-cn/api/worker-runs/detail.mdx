**方法:** `GET`

**端点:** `/api/v2/worker-runs/{runId}`

**认证:** 支持 `Authorization: Bearer <YOUR_API_KEY>`、`api-key: <YOUR_API_KEY>` 和 `?token=<YOUR_API_KEY>`。推荐优先使用 Bearer token。

## 在线试用

<ApiPlayground method="GET" path="/api/v2/worker-runs/{runId}" />

## 什么时候使用这个接口

用于通过已知 `runId` 读取或操作某一次具体运行。

## 标识符说明

- `runId` 是运行记录 slug。启动或重跑后的 `data.run_slug` 就是后续接口使用的 `runId`。

## 路径参数

| 参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| `runId` | 是 | `string` | 运行记录 slug，即启动或重跑响应中的 `data.run_slug`。 |

## 请求示例

```bash
curl -X GET "https://openapi.coreclaw.com/api/v2/worker-runs/YOUR_RUN_ID" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## 注意事项

- API v2 同时支持 Bearer token、旧版 `api-key` 请求头和 query token；新集成建议优先使用 Bearer token。

## HTTP 响应

| HTTP 状态 | 应用代码 | 含义 |
| --- | --- | --- |
| `200` | `0` | 请求成功。 |
| `400` | `11000` | 请求参数不合法。 |
| `401` | `12001` | 认证缺失或无效。 |
| `404` | `11004` | 目标资源不存在。 |
| `422` | `11000` | 请求语义或字段校验未通过。 |
| `429` | `13000` | 请求过于频繁。 |
| `500` | `10000` | 服务端内部错误。 |
