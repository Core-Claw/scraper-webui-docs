**方法:** `GET`

**端点:** `/api/v2/worker-runs/{runId}/result`

**认证:** 支持 `Authorization: Bearer <YOUR_API_KEY>`、`api-key: <YOUR_API_KEY>` 和 `?token=<YOUR_API_KEY>`。推荐优先使用 Bearer token。

## 什么时候使用这个接口

用于分页读取运行结果行，适合结果预览和表格展示。

## 标识符说明

- `runId` 是运行记录 slug。启动或重跑后的 `data.run_slug` 就是后续接口使用的 `runId`。

## 路径参数

| 参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| `runId` | 是 | `string` | 运行记录 slug，即启动或重跑响应中的 `data.run_slug`。 |

## 查询参数

| 参数 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| `offset` | 否 | `integer` | 分页偏移量，从 0 开始；用于结果预览、列表翻页或导出前确认数据窗口。 约束：默认 `0`；最小 0。 |
| `limit` | 否 | `integer` | 每页返回数量；列表和结果接口的 `limit` 上限为 `100`。 约束：默认 `20`；范围 1-100。 |

## 请求示例

```bash
curl -X GET "https://openapi.coreclaw.com/api/v2/worker-runs/YOUR_RUN_ID/result?offset=0&limit=20" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## 响应示例

```json
{
  "code": 0,
  "data": {
    "count": 1,
    "headers": [
      {
        "key": "title",
        "label": "Title"
      }
    ],
    "list": [
      {
        "title": "Example result"
      }
    ],
    "page_index": 1,
    "page_size": 20
  },
  "message": "success",
  "request_id": "req-123"
}
```

## 响应字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `data.count` | `integer` | 当前运行的结果总行数。 |
| `data.headers[]` | `array` | 列定义，每项包含 `key`（字段名）和 `label`（展示名）。 |
| `data.list[]` | `array` | 结果数据行；每行的键对应 `headers[].key`。 |

> 结果行可能包含未在 Worker 输出 schema 中声明的内部字段（例如 `__coreclaw_data_id__`），请按需忽略。嵌套或空值的序列化方式取决于导出格式；`filter_keys` 必须匹配 `headers[].key`。分页请求使用从 0 开始的 `offset` 与 `limit`（最大 `100`）。

## 注意事项

- API v2 同时支持 Bearer token、旧版 `api-key` 请求头和 query token；新集成建议优先使用 Bearer token。
- 结果列表使用 `offset` 和 `limit` 分页。
- `offset` 从 0 开始；`limit` 默认 `20`，最大 `100`。
- 运行状态、轮询、失败诊断和取消处理请参阅[运行生命周期与状态](/zh-cn/api/run-lifecycle/)。

## HTTP 响应

| HTTP 状态 | 应用代码 | 含义 |
| --- | --- | --- |
| `200` | `0` | 请求成功。 |
| `400` | `11000` | 请求参数不合法。 |
| `401` | `12001` | 认证缺失或无效。 |
| `404` | `11004` | 目标资源不存在。 |
| `422` | `11000` | 请求语义或字段校验未通过。 |
| `429` | `13000` | 请求过于频繁。 |
| `500` | `10000` | 服务端内部错误。 |
