**方法:** `GET`

**端点:** `/api/v2/worker-runs/last`

**认证:** 支持 `Authorization: Bearer <YOUR_API_KEY>`、`api-key: <YOUR_API_KEY>` 和 `?token=<YOUR_API_KEY>`。推荐优先使用 Bearer token。

## 什么时候使用这个接口

用于读取或操作当前账户最近一次运行，不限定 Worker。


## 请求示例

```bash
curl -X GET "https://openapi.coreclaw.com/api/v2/worker-runs/last" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## 注意事项

- API v2 同时支持 Bearer token、旧版 `api-key` 请求头和 query token；新集成建议优先使用 Bearer token。
- 这是账户级最近运行接口，不需要 `workerId` 路径参数；它会作用于当前账户范围内最近一次运行。
- `last` 表示当前筛选范围内最近一次运行；如需确定目标，请使用具体 `runId` 接口。

## HTTP 响应

| HTTP 状态 | 应用代码 | 含义 |
| --- | --- | --- |
| `200` | `0` | 请求成功。 |
| `400` | `11000` | 请求参数不合法。 |
| `401` | `12001` | 认证缺失或无效。 |
| `404` | `11004` | 目标资源不存在。 |
| `429` | `13000` | 请求过于频繁。 |
| `500` | `10000` | 服务端内部错误。 |
