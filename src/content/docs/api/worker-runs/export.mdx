**Method:** `GET`

**Endpoint:** `/api/v2/worker-runs/{runId}/result/export`

**Authentication:** Supports `Authorization: Bearer <YOUR_API_KEY>`, `api-key: <YOUR_API_KEY>`, and `?token=<YOUR_API_KEY>`. Prefer Bearer token.

## Try it

<ApiPlayground method="GET" path="/api/v2/worker-runs/{runId}/result/export" />

## When to use this endpoint

Use this endpoint to export run results as a downloadable file.

## Identifier Notes

- `runId` is the run slug. Use `data.run_slug` from start or rerun responses as the `runId`.

## Path Parameters

| Parameter | Required | Type | Description |
| --- | --- | --- | --- |
| `runId` | Yes | `string` | Run slug returned as `data.run_slug` from start or rerun responses. |

## Query Parameters

| Parameter | Required | Type | Description |
| --- | --- | --- | --- |
| `format` | No | `enum: csv, json, jsonl, xlsx, xls, xml, html, rss` | Export format. Supports `csv`, `json`, `jsonl`, `xlsx`, `xls`, `xml`, `html`, `rss` (case-insensitive). Defaults to `csv`. Constraints: default `csv`; allowed values: `csv`, `json`, `jsonl`, `xlsx`, `xls`, `xml`, `html`, `rss`. |
| `filter_keys` | No | `string` | Comma-separated field keys used to limit exported fields, for example `title,address`. |

## Request Example

```bash
curl -X GET "https://openapi.coreclaw.com/api/v2/worker-runs/YOUR_RUN_ID/result/export?format=csv&filter_keys=title%2Caddress" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Response Example

```json
{
  "code": 0,
  "data": {
    "download_url": "https://storage.example.com/runs/01KKDXV2G26BT7NH4ZQR2R4NPZ/results.csv"
  },
  "message": "success",
  "request_id": "req-123"
}
```

## Notes

- API v2 supports Bearer token, the legacy `api-key` header, and query token. Prefer Bearer token for new integrations.
- `filter_keys` can limit exported fields, for example `title,address`.
- Result lists use `offset` and `limit` pagination.
- See [Run Lifecycle & Status](/api/run-lifecycle/) for status handling, polling, failure diagnosis, and cancellation behavior.
