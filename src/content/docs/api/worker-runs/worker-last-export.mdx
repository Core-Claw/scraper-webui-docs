**Method:** `GET`

**Endpoint:** `/api/v2/workers/{workerId}/runs/last/export`

**Authentication:** Supports `Authorization: Bearer <YOUR_API_KEY>`, `api-key: <YOUR_API_KEY>`, and `?token=<YOUR_API_KEY>`. Prefer Bearer token.

## Try it

<ApiPlayground method="GET" path="/api/v2/workers/{workerId}/runs/last/export" />

## When to use this endpoint

Use this endpoint to export run results as a downloadable file.

## Identifier Notes

- `workerId` / `worker_id` accepts a Worker slug or a path encoded as `owner~name` from `owner/name`.

## Path Parameters

| Parameter | Required | Type | Description |
| --- | --- | --- | --- |
| `workerId` | Yes | `string` | Worker slug or path. Encode `owner/name` as `owner~name` for path values. |

## Query Parameters

| Parameter | Required | Type | Description |
| --- | --- | --- | --- |
| `format` | No | `string` | Export format. Defaults to `csv`. Constraints: default `csv`. |
| `filter_keys` | No | `string` | Comma-separated field keys used to limit exported fields, for example `title,address`. |

## Request Example

```bash
curl -X GET "https://openapi.coreclaw.com/api/v2/workers/YOUR_WORKER_ID/runs/last/export?format=csv&filter_keys=title%2Caddress" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Response Example

```json
{
  "code": 0,
  "data": {
    "download_url": "https://storage.example.com/runs/01KKDXV2G26BT7NH4ZQR2R4NPZ/results.csv"
  },
  "message": "success",
  "request_id": "req-123"
}
```

## Notes

- API v2 supports Bearer token, the legacy `api-key` header, and query token. Prefer Bearer token for new integrations.
- This is a Worker-scoped latest-run endpoint; `workerId` is required and the operation applies only to that Worker's latest run.
- `last` means the latest run in the current scope. Use a concrete `runId` endpoint when the target must be explicit.
- `filter_keys` can limit exported fields, for example `title,address`.
